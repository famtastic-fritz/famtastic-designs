# Competitive structure audit

Purpose
- Structural reference only.
- No copied code, copy, assets, logos, testimonials, or metrics.

Reference set
1. Web Design Velocity home page
2. Chetu on-demand developers page
3. QuickPrepAI flow reference requested by owner; live capture was not available during this pass because the domain did not resolve from the proof environment

Observed structural patterns

Web Design Velocity
- Above the fold: loud hero, immediate headline, two CTAs, phone/chat visibility, trust-style visual clutter
- CTA density: very high; repeated Get Started and Live Chat patterns
- Form placement: deferred behind CTA clicks more than embedded in the opening hero
- Pricing/package ladder: category-heavy and menu-heavy rather than clean package proof
- Service taxonomy: broad umbrella sections covering design, development, marketing
- Trust/proof usage: repeated logos, phone number, email, and broad claim language
- Legal/footer: standard footer links, contact channels, and sitemap-like utility
- Mobile implication: heavy nav/CTA repetition can become noisy quickly

Chetu on-demand developers
- Above the fold: category headline + enterprise-style CTA + immediate credibility framing
- CTA density: moderate but persistent through section breaks
- Form placement: CTA-first, with service inventory and capability depth doing the persuasion
- Pricing/package ladder: not public-package based; capability matrix and specialization depth carry the page
- Service taxonomy: deep skill taxonomy and category coverage
- Trust/proof usage: phone numbers, enterprise framing, breadth of technologies
- Legal/footer: typical enterprise footer and utility links
- Mobile implication: long specialist lists require strong section rhythm and anchor discipline

QuickPrepAI
- Owner requested this as a structural reference for contact/service flow.
- Live browser capture was not available in this environment because the domain failed DNS resolution.
- Result: this document does not claim any specific borrowed element from that site beyond the owner’s instruction to think structurally about contact/service flow.

What FAMtastic structurally borrowed
- Clear hero with immediate CTA path
- Repeated contact/get-started opportunities without requiring the visitor to hunt
- Distinct offer ladder rather than one vague “contact us” dead-end
- Service taxonomy broken into understandable categories
- Footer legal/contact utility links

What FAMtastic intentionally avoided
- No copied headline structures
- No fake awards, fake partner logos, or inflated trust badges
- No live-chat theater when chat is not actually staffed
- No enterprise buzzword sprawl
- No placeholder external booking or payment links exposed publicly
- No fake proof metrics

FAMtastic-specific adjustments
- Stronger package ladder tied to actual owner review items
- Forms placed as real conversion surfaces, not decorative placeholders
- Honest proof labels where systems are still mocked
- Admin-editable content path via /admin-proof instead of pretending a full CMS is already active

Copyright boundary

- No copied code
- No copied copy
- No copied design assets
- No copied media
